# 我为什么又自学Python

最近在开始自学 Python，因为有一定的语言基础，所以学起来可能相对比小白来的更快，但是笔记还是要有的，所以在这里进行记录。

熟悉我的读者都知道，我以前自学前端，这一走就是快 3 年了，目前主业是「前端开发」，现在为什么又要去自学 Python 呢？

主要有三个原因吧：

  1. 有时候会有很多想法，想去折腾，但是限制于 JS 不适合，所以想多掌握一门语言。
  2. 语言这么多，为什么就选择 Python 了呢？因为当下 Python 是最热门的语言。而且我本人对数据分析、爬虫感兴趣。刚好 Python 比较适合做这个。所以就选择了它。
  3. 增加自己的技术能力，很多时候在项目开发的过程中，仅仅只能了解前端的一部分，对于整个系统的把控很小。

有人会问，你的前端技术都很牛逼了吗？就去学 Python。

这里说下吧。首先我一直对技术都有敬畏之心，我也不是什么技术大佬。不过我认为，不管是前端也好，后端也罢，亦或者产品，设计等等，最终都是服务于你的目的。而它们都是达到「目的」的工具或者手段而已。

有的人喜欢研究技术把每个细节都掌握，比如成为 Vue 的作者尤雨溪这样的技术大佬。

有的人喜欢掌握更多的工具。比如成为全栈开发工程师，一个人能把一个项目撸完。

这其实都只是选择的不同，并没有什么对错。

如果让我选择的话，我更倾向于后者，因为现在复合型人才会更加有竞争力。

你自己想想看，一个会写文案的销售，一个会技术的产品，一个会写作的技术等等会不会比其他人更具有优势呢？

另外为了防止有人抬杠，我要说下：

你会的多，不一定需要什么都做，那样也不现实，但是你的思维，认知，视野都会被打开，不仅仅限制在自己所熟知的空间里。想想看这种收益是不是值得的。

而我前面也提到了，我是对数据分析感兴趣加上自己也想折腾点东西，所以才选择学习 Python。如果你喜欢在一门技术上深入的研究，那无可厚非嘛。

有的读者会说，登哥你学完 Python 之后会转行吗？

其实不是的，至少目前不会的。我只是在学一门解决问题的工具。为了更方便解决问题，让自己能够随心所欲一点，不会受到限制。

但是，谁也不敢保证以后会怎么样，毕竟能够预测未来的只有神。所以在这里也提醒下大家，那些说自己能够预股票，预测走势基本上都是骗子。大家一定要小心。

不过大家可以放心，我还是会继续分享「前端技术」的，我的主业还是前端开发。后面会给大家分享更多好玩的内容。

今天这篇文章主要是记录下自己现在的想法和观点，等到三年以后在回头看看当时的心境和现在有没有什么变化。

就当作是一个记录吧，看看自己的选择会产生什么样的结果。

加油，如果你也有自己的想法，还在犹犹豫豫，不要在想了，光想是没有结果的，必须要撸起袖子干起来。失败了大不了换个赛道重新跑。你现在多的不只有时间吗？还能损失什么呢？
